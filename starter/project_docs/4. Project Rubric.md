# Rubric

Use this project rubric to understand and assess the project criteria.

## Root Agent

### Criteria

Create and configure a root agent.

### Submission Requirements

- An agent object of type `Agent` is created in the `agent.py` file.
- The agent object is configured with a `name` and `description` suitable for a bird shop customer service agent.
- The agent is configured with an `instruction` parameter that reads from the `agent-prompt.txt` file.
- The agent is configured with all required tools (database, datastore search, and web search).

### Criteria

Configure the agent's session storage and select an appropriate LLM.

### Submission Requirements

- An `InMemorySessionService` is properly imported and instantiated in the `agent.py` file.
- A gemini model is specified for the agent.
- A comment in the `agent.py` file justifies the model selection.

### Criteria

Design and implement an effective agent prompt.

### Submission Requirements

- The agent-prompt.txt file defines a clear persona for the bird shop customer service agent.
- The prompt includes explicit guidelines for when and how the agent should use its tools (database for product pricing, datastore for store info, web search for current trends).
- The prompt sets clear boundaries that the agent should focus on bird-related topics and Betty's Bird Boutique operations.

### Criteria

Demonstrate successful agent execution and tool usage.

### Submission Requirements

- The submission includes screenshots of conversations with the agent showing session IDs.
- Screenshots demonstrate the agent successfully using all three tools (database, datastore search, and web search) in a single conversation.
- For each tool call, screenshots show the detailed request and response, including any tool-specific outputs

## Product databases through MCP Toolbox

### Criteria

Configure database source and implement secure SQL tool.

### Submission Requirements

- The tools.yaml file contains a source with a clear, descriptive name that identifies the data source.
- Source specifies kind: mysql and uses environment variables for host, port, user, and password with ${VAR_NAME} notation.
- Tool has appropriate name with kind: mysql-sql and clear description for querying product pricing.
- Tool has one string parameter with relevant name and description.
 -SQL statement uses LIKE comparison with wildcards and positional parameter (?) to search product names in the products table.

### Criteria

Implement ToolboxSyncClient to load database tool.

### Submission Requirements

- Imports ToolboxSyncClient from toolbox_core.
- Uses os.environ.get("TOOLBOX_URL", "<http://127.0.0.1:5000>") for toolbox URL (no trailing slash).
- Creates ToolboxSyncClient with the URL.
- Loads database tool using db_client.load_tool() and includes in agent's tools list.

### Criteria

Demonstrate successful database tool usage in agent conversations.

### Submission Requirements

- Screenshots show conversations with the agent successfully using the database tool.
- Screenshots capture both user requests for product information and agent responses with database results.
- Tool calls show detailed request and response information including successful product queries and appropriate handling of not-found scenarios.

## Search Tools (Vertex AI & Datastore)

### Criteria

Implement search() function using discoveryengine library with proper configuration and attribution.

### Submission Requirements

- The "datastore.py" file contains updates to the search() function using the discoveryengine library from Google.
- Uses object classes from google.cloud.discoveryengine_v1 library OR documents which alternative classes are used.
- If code is based on Google sample, must state so and explain choices.
- Request object (type SearchRequest) contains minimum attributes: serving config, query parameter, and content search spec of appropriate type.
- Should include other search specification parameters for fine-grained results.

### Criteria

Create properly documented Vertex AI Search Tool function with environment variables and result processing.

### Submission Requirements

- Converts search results to list of strings using appropriate iteration and content extraction.
- Separate tool function calls search function with appropriate parameters and returns processed results.
- Tool definition uses docstring comments.
- Project ID, engine ID, and location read from environment variables with appropriate defaults.
- Tool function takes search query parameter and calls search with environment configuration values.

### Criteria

Code demonstrating datastore tool integration

### Submission Requirements

- The code in "agent.py" imports the tool defined in "datastore.py" and makes this tool available to the root agent.
- The datastore search tool must be defined in the same file as search() which should be "datastore.py".
- The "agent.py" file must import the tool function from "datastore.py" and include it in the list of tools to be sent to the root agent constructor.

### Criteria

Demonstrate operation of the datastore tool as part of a conversation with the agent.

### Submission Requirements

- Screenshots from conversations with the agent illustrating successful tool calls.
- Screenshots should capture details of both the request and the response.
- The response is not expected to answer the question, but should contain part of the source document that will.

## Grounding with Google Search

### Criteria

Code demonstrating an agent tool that uses Grounding with Google Search to answer questions and correct the incorporation of this tool as part of the root agent configuration.

### Submission Requirements

- Code in "search_agent.py" file creates an Agent object with appropriate name and description for search functionality.
- Agent includes google_search tool imported from google.adk.tools in its tools list.
- Creates AgentTool wrapper around the search agent for integration with root agent.
- Root agent imports and includes the search agent tool in its tools list.

### Criteria

Demonstrate evaluation and selection of an appropriate model and prompt for the search agent.

### Submission Requirements

- Search agent uses an appropriate Gemini 2.0 or 2.5 family model with comments explaining the choice.
- Model selection considers performance requirements for search functionality.
- The prompt in "search-prompt.txt" contains an appropriate prompt for the search agent's role.
- Prompt design is suitable for handling search-related queries.

### Criteria

Demonstrate operation of the search tool as part of a conversation with the agent.

### Submission Requirements

- Screenshots from conversations with the agent illustrating successful searches using the Google Search grounding tool.
- Screenshots should capture details of both the request and the response.
- Evidence shows the agent using web search to answer questions requiring external or real-time information.
- Search responses include visible attribution or sources indicating grounded search results.

## General coding best practices

### Criteria

Demonstrate proper division of tasks into different tools and splitting of tools into appropriate functions.

### Submission Requirements

- Tool-related functions are defined in separate files (e.g., datastore.py, search_agent.py) and imported into agent.py.
- Helper functions are co-located in the same file as the tool functions that use them.
- Variable and function names are descriptive and follow a discernible naming convention (e.g., snake_case).
- The code includes comments and docstrings at appropriate places.

## Suggestions to Make Your Project Stand Out

1. **Expand Your Knowledge Base**

    Consider adding more documents that describe the store. What other information might customers want to know about the store or birds?

    Some suggestions to try:

    - Return policy
    - Special holiday hours
    - Promotions or sales
    - Staff bios (insert yourself!)

    _Test conversations both before and after adding new information to see how the agent's responses improve_.

2. **Add Date Awareness**

    Although you have store hours, the agent doesn't know today's date, so it can't answer "Are you open today?". **Create a tool** that provides the current date and day of the week**.**

3. **Improve Product Matching**

    The database currently requires fairly exact product name matching. You can improve this in two ways:

    - `Vertex AI Search` approach: Use Vertex AI Search with structured data.
    - Product list approach: Create a tool that retrieves all product names and embeds them in the conversation for similarity matching.

    Try both approaches and compare:

    - Which gives better results with typos or partial names?
    - What are the performance trade-offs?
    - How do they handle edge cases?
